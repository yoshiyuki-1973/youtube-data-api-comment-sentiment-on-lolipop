# 要件定義書

YouTube動画コメント感情分析システム（PHP / ロリポップ版）

## 1. 目的

YouTube Data API v3 を利用して動画メタデータおよびコメントを取得し、Grok API（xAI）による感情分析（ポジティブ / ネガティブ / ニュートラル）を実行し、集計結果をブラウザのWebアプリで表示するシステムである。

分析結果は MySQL に24時間キャッシュし、同一条件での再アクセス時にはAPI呼び出しを省略して即座に返す。

## 2. 対象範囲

- 動画メタデータ取得（再生回数、高評価数、コメント数など）
- コメント取得（いいね数順ソート対応）
- Grok APIによる感情分析（10件バッチ処理）
- 集計・結果表示（ブラウザ上のドーナツグラフ + コメント一覧）
- MySQLキャッシュ（24時間保持）

## 3. 機能要件

### 3.1 動画メタデータ取得

YouTube Data API v3 の `videos.list` を使用し、動画IDごとに以下のメタデータを取得する。

| 項目 | APIフィールド | 型 |
|------|-------------|-----|
| 動画ID | id | string |
| タイトル | snippet.title | string |
| チャンネルID | snippet.channelId | string |
| チャンネル名 | snippet.channelTitle | string |
| 公開日 | snippet.publishedAt | ISO 8601 |
| 再生回数 | statistics.viewCount | integer |
| 高評価数 | statistics.likeCount | integer |
| コメント数 | statistics.commentCount | integer |

### 3.2 コメント取得

- 取得件数: 10 / 30 / 50 / 100件（画面のセレクトボックスで選択）
- 取得順序: `relevance` 順で取得後、いいね数降順にソート
- ソート最適化: 指定件数の2倍（最大100件）を取得してメモリ上でソートし、上位を返す

> UI では `10 / 30 / 50 / 100` の固定選択肢を提供する。
> 一方でサーバー側 API はバリデーション上 `1〜100` を受け付ける。

### 3.3 感情分析

Grok API（`grok-3-mini`）を使用し、10件ずつバッチ処理で感情分析を実行する。

| 分類ラベル | 意味 | スコア範囲 |
|-----------|------|-----------|
| `pos` | ポジティブ | 0.0 - 1.0 |
| `neg` | ネガティブ | 0.0 - 1.0 |
| `neutral` | 中立/判定困難 | 0.0 - 1.0 |

- 3つのスコアの合計は常に 1.0（正規化済み）
- 最大スコアのラベルを感情ラベルとして採用

### 3.4 集計・表示

動画単位でコメントの感情分析結果を集計し、以下を表示する。

- 感情分類別件数（positive / negative / neutral）
- 感情分類比率
- 平均スコア（avg_positive_score / avg_negative_score / avg_neutral_score）
- ドーナツグラフ（Chart.js 4.4.0）
- コメント一覧（感情フィルタタブ付き）

### 3.5 WebUI

| 機能 | 説明 |
|------|------|
| 動画ID/URL入力 | 複数形式のYouTube URL対応 |
| コメント件数選択 | 10 / 30 / 50 / 100件をセレクトボックスで選択 |
| 分析実行 | ボタンクリックで非同期（AJAX）分析開始 |
| 動画情報表示 | タイトル・チャンネル名・再生回数・高評価数・コメント数 |
| 感情グラフ | Chart.js ドーナツグラフ（ポジ/ネガ/ニュートラル比率） |
| コメント一覧 | 感情フィルタタブ（すべて / ポジ / ネガ / ニュートラル）付き |
| エラーハンドリング | API制限・認証エラー等の日本語表示 |

対応URL形式:
- `https://www.youtube.com/watch?v=VIDEO_ID`
- `https://youtu.be/VIDEO_ID`
- `https://www.youtube.com/shorts/VIDEO_ID`
- `https://www.youtube.com/embed/VIDEO_ID`
- 11文字の動画ID直接入力

### 3.6 MySQLキャッシュ

- 動画ID × コメント件数 の組み合わせで24時間キャッシュ
- キャッシュヒット時はYouTube API・Grok APIを呼び出さずに即返却
- キャッシュデータはphpMyAdminから手動削除可能

> `動画ID × コメント件数` は **UNIQUE 制約ではなくキャッシュ検索キー** として扱う。
> 同じ組み合わせの履歴が複数保存されることがあり、その場合は `analyzed_at` が最も新しいレコードを採用する。

## 4. 非機能要件

### 4.1 実行環境

- ロリポップ共有サーバー（ハイスピードプラン）
- PHP 8.1以上（Apache + mod_php）
- MySQL 5.7以上（ロリポップ マイSQL）
- Docker 不使用

### 4.2 認証・セキュリティ

- YouTube API キーおよび Grok API キーは `config.php` で管理（`.gitignore` 対象・Web非公開）
- `src/`・`sql/`・`config.php` は `.htaccess` でHTTPアクセスを禁止
- SQLインジェクション対策: PDO プリペアドステートメント
- XSS対策: フロントエンドで `escapeHtml()` 実装
- `api/analyze.php` は POST リクエストのみ受け付ける

### 4.3 実行方式

- 手動実行（スケジューラ不使用）
- ブラウザからの対話的操作のみ（CLIバッチ処理なし）

### 4.4 データ永続化

- 分析結果（動画情報・感情スコア・コメント）を MySQL に保存
- キャッシュ有効期間: 24時間
- ログはロリポップのサーバーログを使用（`error_log()` で出力）

### 4.5 パフォーマンス目標

| 条件 | 目標応答時間 |
|------|------------|
| キャッシュヒット | 1秒以内 |
| 10件分析（初回） | 約2〜5秒 |
| 50件分析（初回） | 約10〜25秒 |
| 100件分析（初回） | 約20〜50秒 |

## 5. APIクォータ対策

### 5.1 YouTube Data API v3 のクォータ制限

- 1日あたり10,000ユニット（無料枠）
- 主なコスト:
  - `videos.list`: 1ユニット
  - `commentThreads.list`: 1ユニット（ページあたり）

### 5.2 対策方針

1. **24時間キャッシュ**: 同一動画を再分析してもAPIを呼び出さない
2. **件数制限**: 1リクエストあたり最大100件（API上限）
3. **エラーハンドリング**: クォータ超過（403 quotaExceeded）時は HTTP 429 を返してユーザーに通知

### 5.3 Grok API のレート制限対策

- HTTP 429 返却時は `GrokApiException` で通知
- リトライは実装しない（ユーザーが数分後に再試行）
