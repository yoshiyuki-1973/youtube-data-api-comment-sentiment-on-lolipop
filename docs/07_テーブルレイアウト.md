# テーブルレイアウト

YouTube動画コメント感情分析システム - MySQLテーブル定義書

**DB**: ロリポップ マイSQL（MySQL 5.7以上）
**文字コード**: utf8mb4 / utf8mb4_unicode_ci
**スキーマファイル**: `php/sql/schema.sql`

---

## テーブル一覧

| # | テーブル名 | 論理名 | 説明 |
|---|-----------|--------|------|
| 1 | `videos` | 動画マスタ | YouTube動画のメタデータキャッシュ |
| 2 | `analysis_results` | 分析結果 | 感情分析の集計結果（動画×取得件数ごと） |
| 3 | `comments` | コメント | 分析済みコメント詳細 |

**リレーション**: `videos` 1:N `analysis_results` 1:N `comments`

---

## 1. videos（動画マスタ）

YouTube動画のメタデータをキャッシュするテーブル。同じ動画IDへのAPIリクエストを削減する。

### カラム定義

| カラム名 | 型 | NULL | デフォルト | PK/FK/UQ | 説明 |
|---------|-----|------|-----------|----------|------|
| id | INT UNSIGNED | NOT NULL | AUTO_INCREMENT | PK | サロゲートキー |
| video_id | VARCHAR(20) | NOT NULL | - | UQ | YouTube動画ID（11文字） |
| title | VARCHAR(500) | NOT NULL | - | | 動画タイトル |
| channel_name | VARCHAR(255) | NOT NULL | - | | チャンネル名 |
| channel_id | VARCHAR(50) | NOT NULL | - | | チャンネルID |
| published_at | DATETIME | NOT NULL | - | | 動画公開日時（UTC） |
| view_count | BIGINT UNSIGNED | NOT NULL | 0 | | 再生回数 |
| like_count | BIGINT UNSIGNED | NOT NULL | 0 | | 高評価数 |
| comment_count | BIGINT UNSIGNED | NOT NULL | 0 | | コメント総数 |
| fetched_at | DATETIME | NOT NULL | - | | APIから取得した日時（UTC） |

### インデックス

| 名前 | カラム | 種別 | 目的 |
|------|--------|------|------|
| PRIMARY | id | PRIMARY KEY | - |
| (UNIQUE) | video_id | UNIQUE | 重複防止・高速検索 |
| idx_fetched_at | fetched_at | INDEX | キャッシュ期限チェック |

### DDL

```sql
CREATE TABLE IF NOT EXISTS videos (
    id            INT UNSIGNED    AUTO_INCREMENT PRIMARY KEY,
    video_id      VARCHAR(20)     NOT NULL UNIQUE,
    title         VARCHAR(500)    NOT NULL,
    channel_name  VARCHAR(255)    NOT NULL,
    channel_id    VARCHAR(50)     NOT NULL,
    published_at  DATETIME        NOT NULL,
    view_count    BIGINT UNSIGNED NOT NULL DEFAULT 0,
    like_count    BIGINT UNSIGNED NOT NULL DEFAULT 0,
    comment_count BIGINT UNSIGNED NOT NULL DEFAULT 0,
    fetched_at    DATETIME        NOT NULL,
    INDEX idx_fetched_at (fetched_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### データ例

| id | video_id | title | channel_name | view_count | fetched_at |
|----|----------|-------|--------------|------------|------------|
| 1 | dQw4w9WgXcQ | Never Gonna Give You Up | Rick Astley | 1400000000 | 2024-01-15 10:30:00 |

## 2. analysis_results（分析結果）

感情分析の集計結果を保存するテーブル。**同じ動画ID × 同じ取得件数**をキャッシュ検索単位として24時間キャッシュする。
ただし、`video_id × comment_limit` に **一意制約は設けていない**。同じ組み合わせでも分析時刻の異なる履歴を複数保持でき、キャッシュ判定時は `analyzed_at` の新しいレコードを優先して参照する。

### カラム定義

| カラム名 | 型 | NULL | デフォルト | PK/FK | 説明 |
|---------|-----|------|-----------|-------|------|
| id | INT UNSIGNED | NOT NULL | AUTO_INCREMENT | PK | サロゲートキー |
| video_id | VARCHAR(20) | NOT NULL | - | FK → videos.video_id | YouTube動画ID |
| comment_limit | SMALLINT UNSIGNED | NOT NULL | 10 | | 取得コメント件数（1〜100） |
| positive_count | SMALLINT UNSIGNED | NOT NULL | 0 | | ポジティブ件数 |
| negative_count | SMALLINT UNSIGNED | NOT NULL | 0 | | ネガティブ件数 |
| neutral_count | SMALLINT UNSIGNED | NOT NULL | 0 | | ニュートラル件数 |
| total_count | SMALLINT UNSIGNED | NOT NULL | 0 | | 分析総件数 |
| positive_ratio | DECIMAL(5,4) | NOT NULL | 0.0000 | | ポジティブ比率（0.0000〜1.0000） |
| negative_ratio | DECIMAL(5,4) | NOT NULL | 0.0000 | | ネガティブ比率 |
| neutral_ratio | DECIMAL(5,4) | NOT NULL | 0.0000 | | ニュートラル比率 |
| avg_positive_score | DECIMAL(5,4) | NOT NULL | 0.0000 | | 平均ポジティブスコア |
| avg_negative_score | DECIMAL(5,4) | NOT NULL | 0.0000 | | 平均ネガティブスコア |
| avg_neutral_score | DECIMAL(5,4) | NOT NULL | 0.0000 | | 平均ニュートラルスコア |
| analyzed_at | DATETIME | NOT NULL | - | | 分析実行日時（UTC） |

### インデックス

| 名前 | カラム | 種別 | 目的 |
|------|--------|------|------|
| PRIMARY | id | PRIMARY KEY | - |
| idx_video_limit | (video_id, comment_limit) | INDEX | キャッシュ検索（動画ID + 件数）。一意制約ではない |
| idx_analyzed_at | analyzed_at | INDEX | キャッシュ期限チェック（24h） |
| FK | video_id → videos.video_id | FOREIGN KEY | 参照整合性（CASCADE DELETE） |

### DDL

```sql
CREATE TABLE IF NOT EXISTS analysis_results (
    id                  INT UNSIGNED   AUTO_INCREMENT PRIMARY KEY,
    video_id            VARCHAR(20)    NOT NULL,
    comment_limit       SMALLINT UNSIGNED NOT NULL DEFAULT 10,
    positive_count      SMALLINT UNSIGNED NOT NULL DEFAULT 0,
    negative_count      SMALLINT UNSIGNED NOT NULL DEFAULT 0,
    neutral_count       SMALLINT UNSIGNED NOT NULL DEFAULT 0,
    total_count         SMALLINT UNSIGNED NOT NULL DEFAULT 0,
    positive_ratio      DECIMAL(5,4)   NOT NULL DEFAULT 0.0000,
    negative_ratio      DECIMAL(5,4)   NOT NULL DEFAULT 0.0000,
    neutral_ratio       DECIMAL(5,4)   NOT NULL DEFAULT 0.0000,
    avg_positive_score  DECIMAL(5,4)   NOT NULL DEFAULT 0.0000,
    avg_negative_score  DECIMAL(5,4)   NOT NULL DEFAULT 0.0000,
    avg_neutral_score   DECIMAL(5,4)   NOT NULL DEFAULT 0.0000,
    analyzed_at         DATETIME       NOT NULL,
    FOREIGN KEY (video_id) REFERENCES videos(video_id) ON DELETE CASCADE,
    INDEX idx_video_limit (video_id, comment_limit),
    INDEX idx_analyzed_at (analyzed_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### データ例

| id | video_id | comment_limit | positive_count | negative_count | neutral_count | total_count | positive_ratio | analyzed_at |
|----|----------|--------------|---------------|---------------|--------------|-------------|----------------|-------------|
| 1 | dQw4w9WgXcQ | 10 | 7 | 2 | 1 | 10 | 0.7000 | 2024-01-15 10:30:05 |

## 3. comments（コメント）

分析済みコメントの詳細を保存するテーブル。analysis_results に紐づく。

### カラム定義

| カラム名 | 型 | NULL | デフォルト | PK/FK | 説明 |
|---------|-----|------|-----------|-------|------|
| id | INT UNSIGNED | NOT NULL | AUTO_INCREMENT | PK | サロゲートキー |
| analysis_id | INT UNSIGNED | NOT NULL | - | FK → analysis_results.id | 分析結果ID |
| comment_text | TEXT | NOT NULL | - | | コメント本文（最大65,535バイト） |
| author_name | VARCHAR(255) | NOT NULL | '' | | 投稿者名 |
| like_count | INT UNSIGNED | NOT NULL | 0 | | コメントへのいいね数 |
| published_at | DATETIME | NOT NULL | - | | コメント投稿日時（UTC） |
| sentiment | ENUM('pos','neg','neutral') | NOT NULL | 'neutral' | | 感情ラベル |
| positive_score | DECIMAL(5,4) | NOT NULL | 0.0000 | | ポジティブ確率スコア（0〜1） |
| negative_score | DECIMAL(5,4) | NOT NULL | 0.0000 | | ネガティブ確率スコア |
| neutral_score | DECIMAL(5,4) | NOT NULL | 0.0000 | | ニュートラル確率スコア |

### インデックス

| 名前 | カラム | 種別 | 目的 |
|------|--------|------|------|
| PRIMARY | id | PRIMARY KEY | - |
| idx_analysis_sentiment | (analysis_id, sentiment) | INDEX | 感情フィルタ付きコメント取得 |
| FK | analysis_id → analysis_results.id | FOREIGN KEY | 参照整合性（CASCADE DELETE） |

### DDL

```sql
CREATE TABLE IF NOT EXISTS comments (
    id              INT UNSIGNED   AUTO_INCREMENT PRIMARY KEY,
    analysis_id     INT UNSIGNED   NOT NULL,
    comment_text    TEXT           NOT NULL,
    author_name     VARCHAR(255)   NOT NULL DEFAULT '',
    like_count      INT UNSIGNED   NOT NULL DEFAULT 0,
    published_at    DATETIME       NOT NULL,
    sentiment       ENUM('pos','neg','neutral') NOT NULL DEFAULT 'neutral',
    positive_score  DECIMAL(5,4)   NOT NULL DEFAULT 0.0000,
    negative_score  DECIMAL(5,4)   NOT NULL DEFAULT 0.0000,
    neutral_score   DECIMAL(5,4)   NOT NULL DEFAULT 0.0000,
    FOREIGN KEY (analysis_id) REFERENCES analysis_results(id) ON DELETE CASCADE,
    INDEX idx_analysis_sentiment (analysis_id, sentiment)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### データ例

| id | analysis_id | comment_text | author_name | like_count | sentiment | positive_score | negative_score | neutral_score |
|----|-------------|-------------|-------------|------------|-----------|----------------|----------------|---------------|
| 1 | 1 | 最高の動画です！ | ユーザーA | 42 | pos | 0.9200 | 0.0500 | 0.0300 |
| 2 | 1 | 時間の無駄だった | ユーザーB | 8 | neg | 0.0800 | 0.8700 | 0.0500 |

## テーブル間リレーション

```
videos
  │ video_id (UNIQUE)
  │ 1
  │
  ↓ N
analysis_results
  │ id (PK)
  │ video_id (FK)
  │ comment_limit
  │ 1
  │
  ↓ N
comments
    id (PK)
    analysis_id (FK)
    sentiment
    *_score
```

### CASCADE DELETEの挙動

| 操作 | 結果 |
|------|------|
| `videos` のレコード削除 | 紐づく `analysis_results` と `comments` も自動削除 |
| `analysis_results` のレコード削除 | 紐づく `comments` も自動削除 |

## キャッシュ設計

- **キャッシュ有効期間**: 24時間（`VideoRepository::CACHE_HOURS = 24`）
- **キャッシュキー**: `video_id` × `comment_limit` の組み合わせ
- **一意性の扱い**: `video_id` × `comment_limit` は検索キーであり、UNIQUE ではない
- **キャッシュヒット条件**: `analyzed_at > NOW() - 24時間`
- **採用レコード**: 条件を満たす中で `analyzed_at` が最も新しい1件
- **キャッシュミス時**: YouTube API + Grok API を呼び出して新規保存

```sql
-- キャッシュ検索クエリ（VideoRepository::findCachedResult）
SELECT ar.*, v.*
FROM analysis_results ar
JOIN videos v ON v.video_id = ar.video_id
WHERE ar.video_id = :video_id
  AND ar.comment_limit = :comment_limit
  AND ar.analyzed_at > DATE_SUB(NOW(), INTERVAL 24 HOUR)
ORDER BY ar.analyzed_at DESC
LIMIT 1;
```
