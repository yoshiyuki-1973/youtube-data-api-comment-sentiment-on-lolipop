# 詳細設計書

YouTube動画コメント感情分析システム（PHP / ロリポップ版）

## 1. 設計方針

- 各クラスに単一責任を持たせ、疎結合に設計する
- 外部依存（YouTube API / Gemini API / MySQL）はクラスに閉じ込め、テスト時はモック可能にする
- PDO プリペアドステートメントで SQL インジェクション対策を徹底する
- エラーはカスタム例外で表現し、APIエンドポイントで HTTP ステータスにマッピングする

---

## 2. ディレクトリ構成（php/src/）

```
php/src/
├── bootstrap.php         # spl_autoload_register + Config::load()
├── Config.php            # 設定値ローダー
├── Database.php          # PDO シングルトン
├── YouTubeClient.php     # YouTube Data API クライアント
├── GeminiClient.php      # Gemini API クライアント（バッチ処理）
├── SentimentAnalyzer.php # 感情スコア付与・集計
├── VideoRepository.php   # MySQL CRUD・キャッシュ管理
└── Exceptions.php        # カスタム例外クラス群
```

## 3. クラス仕様

### 3.1 Config クラス

**ファイル**: `src/Config.php`

```php
class Config
{
    public static function load(string $configFile): void
    public static function get(string $key, mixed $default = null): mixed
    public static function require(string $key): string
}
```

| メソッド | 説明 |
|---------|------|
| `load(string $configFile): void` | `config.php` を読み込む。二重ロード防止済み。ファイル不存在時は `RuntimeException` |
| `get(string $key, mixed $default = null): mixed` | 設定値を返す。キー不存在時は `$default` を返す |
| `require(string $key): string` | 設定値を返す。キー不存在または空文字の場合は `RuntimeException` |

**bootstrap.php での初期化**:
```php
// spl_autoload_register で src/ 配下のクラスをオートロード
// Config::load($configFile) で設定を読み込む
```

**config.php の期待形式**:
```php
return [
    'YOUTUBE_API_KEY' => '...',
    'GEMINI_API_KEY'  => '...',
    'GEMINI_MODEL'    => 'gemini-2.5-flash',
    'DB_HOST'         => 'mysql***.lolipop.jp',
    'DB_NAME'         => '...',
    'DB_USER'         => '...',
    'DB_PASS'         => '...',
];
```

### 3.3 YouTubeClient クラス

**ファイル**: `src/YouTubeClient.php`

```php
class YouTubeClient
{
    public function __construct(string $apiKey)

    public static function extractVideoId(string $input): string

    public function fetchVideo(string $videoId): array

    public function fetchComments(string $videoId, int $limit = 10): array
}
```

#### extractVideoId

```php
public static function extractVideoId(string $input): string
```

| 入力形式 | 例 |
|---------|-----|
| 11文字の動画ID | `dQw4w9WgXcQ` |
| `watch?v=` URL | `https://www.youtube.com/watch?v=dQw4w9WgXcQ` |
| `youtu.be/` 短縮URL | `https://youtu.be/dQw4w9WgXcQ` |
| `/shorts/` URL | `https://www.youtube.com/shorts/dQw4w9WgXcQ` |
| `/embed/` URL | `https://www.youtube.com/embed/dQw4w9WgXcQ` |

- 有効な動画IDが取得できない場合は `\InvalidArgumentException` を送出（`VideoNotFoundException` ではない）

#### fetchVideo

```php
public function fetchVideo(string $videoId): array
```

**戻り値**:
```php
[
    'video_id'      => string,
    'title'         => string,
    'channel_name'  => string,
    'channel_id'    => string,
    'published_at'  => string,  // ISO 8601
    'view_count'    => int,
    'like_count'    => int,
    'comment_count' => int,
]
```

**例外**:
| 条件 | 例外 |
|------|------|
| items が空 | `VideoNotFoundException` |
| 403 quotaExceeded | `QuotaExceededException` |
| 403 commentsDisabled | `CommentsDisabledException` |
| 403 その他 | `AuthenticationException` |
| 404 | `VideoNotFoundException` |
| その他 HTTP エラー | `YouTubeApiException` |
| curl 失敗 | `YouTubeApiException` |

#### fetchComments

```php
public function fetchComments(string $videoId, int $limit = 10): array
```

**戻り値の各要素**:
```php
[
    'text'         => string,
    'author'       => string,   // ← 'author_name' ではなく 'author'
    'like_count'   => int,
    'published_at' => string,   // ISO 8601
]
```

**前処理**:
- `textDisplay` を取得した後、Gemini API に渡す直前に `strip_tags()` でHTMLタグを除去する
- 感情分析用プロンプトには先頭300文字までを使用する
- 元のコメント本文は `text` フィールドとして保持し、表示用データにはそのまま含める

**例外**: `fetchVideo` と同じ例外クラスを使用

### 3.4 GeminiClient クラス

**ファイル**: `src/GeminiClient.php`

```php
class GeminiClient
{
    private const BATCH_SIZE = 20;
    private const TIMEOUT = 60;

    public function __construct(string $apiKey, string $model = 'gemini-2.5-flash')

    public function analyzeSentimentBatch(array $comments): array
}
```

#### analyzeSentimentBatch

```php
public function analyzeSentimentBatch(array $comments): array
```

- `$comments` が空の場合は空配列を返す（API呼び出しなし）
- `BATCH_SIZE`（20件）ずつに分割してバッチ処理
- 各バッチで `POST https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent` を呼び出す
- レスポンスの JSON 配列から `positive` / `negative` / `neutral` を抽出・正規化
- JSON 解析失敗 / 件数不足の場合はデフォルトスコア `[0.33, 0.33, 0.34]` で補完

**戻り値の各要素**:
```php
[
    'positive' => float,  // 0.0 - 1.0
    'negative' => float,  // 0.0 - 1.0
    'neutral'  => float,  // 0.0 - 1.0
]
```

**例外**:
| 条件 | 例外 |
|------|------|
| HTTP 429 | `GeminiApiException` |
| HTTP 200以外 | `GeminiApiException` |
| curl 失敗 | `GeminiApiException` |

### 3.5 SentimentAnalyzer クラス

**ファイル**: `src/SentimentAnalyzer.php`

```php
class SentimentAnalyzer
{
    public function __construct(GeminiClient $geminiClient)

    public function analyzeComments(array $comments): array

    public static function summarize(array $analyzed): array

    // determineSentiment は private メソッド（外部から直接呼び出し不可）
    private function determineSentiment(array $score): string
}
```

#### analyzeComments

```php
public function analyzeComments(array $comments): array
```

- コメント配列を `GeminiClient::analyzeSentimentBatch()` に渡す
- 各コメントに `sentiment` / `positive_score` / `negative_score` / `neutral_score` を追加して返す

**戻り値の各要素**（入力のフィールドに以下を追加）:
```php
[
    // 入力フィールドをすべて保持したうえで以下を追加
    'sentiment'      => string,  // 'pos' | 'neg' | 'neutral'
    'positive_score' => float,
    'negative_score' => float,
    'neutral_score'  => float,
]
```

#### summarize

```php
public static function summarize(array $analyzed): array
```

**戻り値**（キー名は `avg_positive_score` 系 — `avg_positive` ではない）:
```php
[
    'positive_count'     => int,
    'negative_count'     => int,
    'neutral_count'      => int,
    'total_count'        => int,
    'positive_ratio'     => float,  // 4桁精度
    'negative_ratio'     => float,
    'neutral_ratio'      => float,
    'avg_positive_score' => float,  // ← 'avg_positive' ではない
    'avg_negative_score' => float,
    'avg_neutral_score'  => float,
]
```

#### determineSentiment（private）

同値の場合の優先順: `pos` > `neutral` > `neg`

| 条件 | 戻り値 |
|------|--------|
| positive ≥ negative かつ positive ≥ neutral | `'pos'` |
| negative > positive かつ negative ≥ neutral | `'neg'` |
| それ以外 | `'neutral'` |

### 3.6 VideoRepository クラス

**ファイル**: `src/VideoRepository.php`

```php
class VideoRepository
{
    public function __construct(PDO $db)

    public function findCachedResult(string $videoId, int $commentLimit): ?array

    public function saveResult(array $video, array $analyzedComments, array $summary, int $commentLimit): array
}
```

#### findCachedResult

```php
public function findCachedResult(string $videoId, int $commentLimit): ?array
```

- `analyzed_at > DATE_SUB(NOW(), INTERVAL 24 HOUR)` のレコードを検索
- ヒットした場合は動画情報 + 集計サマリー + コメント配列が混在したフラット配列を返す
- ヒットしない場合は `null` を返す

#### saveResult

```php
public function saveResult(array $video, array $analyzedComments, array $summary, int $commentLimit): array
```

- 引数の順序: `(video, analyzedComments, summary, commentLimit)` ← summaryとcommentsの順序に注意
- 戻り値の型: `array`（保存後の動画情報 + サマリー + コメント配列）
- トランザクション内で実行:
  1. `videos` テーブルを `video_id` で UPSERT
  2. `analysis_results` テーブルに挿入 → `lastInsertId` を取得
  3. `comments` テーブルに一括挿入
- 失敗時はロールバックして例外を再送出

## 4. api/analyze.php 仕様

**ファイル**: `api/analyze.php`

- POST リクエストのみ受け付ける（それ以外は 405 を返す）
- Content-Type: `application/json; charset=utf-8` を返す
- `src/bootstrap.php` で `spl_autoload_register` と `Config::load()` を初期化

**リクエストボディ**（JSON）:
```json
{
  "url": "YouTube URL または 動画ID",
  "limit": 50
}
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `url` | string | YouTube URL または 11文字の動画ID（**`input` ではなく `url`**） |
| `limit` | int | コメント取得件数（1〜100、省略時は10）（**`comment_limit` ではなく `limit`**） |

> UI では `10 / 30 / 50 / 100` の固定選択肢を表示するが、API 層では `1〜100` を受け付ける。
> 固定選択肢は画面仕様、`1〜100` はサーバー側バリデーション仕様である。

**レスポンス（成功）**:
```json
{
  "success": true,
  "cached": false,
  "video": {
    "video_id": "string",
    "title": "string",
    "channel_name": "string",
    "view_count": 0,
    "like_count": 0,
    "comment_count": 0
  },
  "summary": {
    "positive_count": 0,
    "negative_count": 0,
    "neutral_count": 0,
    "total_count": 0,
    "positive_ratio": 0.0,
    "negative_ratio": 0.0,
    "neutral_ratio": 0.0,
    "avg_positive_score": 0.0,
    "avg_negative_score": 0.0,
    "avg_neutral_score": 0.0
  },
  "comments": [
    {
      "text": "string",
      "author": "string",
      "like_count": 0,
      "published_at": "string",
      "sentiment": "pos",
      "positive_score": 0.0,
      "negative_score": 0.0,
      "neutral_score": 0.0
    }
  ]
}
```

> `cached: true` の場合は YouTube API / Gemini API の呼び出しなし。キャッシュヒット時も同じ JSON 形式で返す。

**レスポンス（エラー）**（`code` フィールドは存在しない）:
```json
{
  "success": false,
  "error": "エラーメッセージ（日本語）"
}
```

**HTTP ステータスコードマッピング**:

| 例外クラス | HTTP | エラーメッセージ |
|-----------|------|----------------|
| `\InvalidArgumentException` | 400 | 入力値のエラーメッセージそのまま |
| `VideoNotFoundException` | 404 | 例外メッセージそのまま |
| `QuotaExceededException` | 429 | 例外メッセージそのまま |
| `AuthenticationException` | 403 | 例外メッセージそのまま |
| `CommentsDisabledException` | 422 | 例外メッセージそのまま |
| `GeminiApiException` | 502 | 「AI分析サービスへの接続に失敗しました。しばらく待ってから再試行してください。」 |
| その他の例外 | 500 | 「サーバーエラーが発生しました。」（詳細は `error_log` のみ） |

## 5. 命名規約

### 5.1 識別子

| 種類 | 規則 | 例 |
|------|------|-----|
| クラス名 | PascalCase | `YouTubeClient`, `GeminiClient` |
| メソッド名 | camelCase | `fetchVideo()`, `analyzeComments()` |
| プロパティ名 | camelCase | `$apiKey`, `$model` |
| 定数 | UPPER_SNAKE_CASE | `BATCH_SIZE`, `TIMEOUT` |
| DBカラム名 | snake_case | `video_id`, `analyzed_at` |
| JSONキー | snake_case | `video_id`, `positive_score` |

### 5.2 感情ラベル

| 内部ラベル | 表示名 | 説明 |
|-----------|--------|------|
| `pos` | ポジティブ | 肯定的なコメント |
| `neg` | ネガティブ | 否定的なコメント |
| `neutral` | ニュートラル | 中立・判定困難 |
