# ディレクトリ構成

YouTube コメント感情分析システムのディレクトリ構成を示す。  
現行システムは **ロリポップ共有サーバーへ FTP で配置する PHP アプリケーション** であり、旧 Python / Docker 構成は削除済みである。

---

## 1. ルート構成

```text
youtube-data-api-comment-sentiment-on-lolipop/
├── php/                        # メインアプリケーション
├── docs/                       # 設計・運用ドキュメント
├── .gitignore                  # Git 除外設定
├── README.md                   # プロジェクト概要
└── CLAUDE.md                   # 開発補助ガイド
```

---

## 2. php/ - メインアプリケーション

```text
php/
├── index.php                   # メイン画面（HTML + AJAX 起点）
├── config.php.example          # 設定サンプル（Git 管理対象）
├── config.php                  # 実設定ファイル（Git 管理外）
├── composer.json               # Composer 設定
├── composer.lock               # Composer ロックファイル
├── phpunit.xml                 # PHPUnit 設定
├── .htaccess                   # src/ / sql/ / config.php への直接アクセス制御
├── api/
│   └── analyze.php             # AJAX エンドポイント
├── src/
│   ├── Config.php              # 設定ロード
│   ├── Database.php            # PDO 接続
│   ├── Exceptions.php          # カスタム例外
│   ├── GrokClient.php          # Grok API 呼び出し
│   ├── SentimentAnalyzer.php   # 感情分析集計
│   ├── VideoRepository.php     # MySQL CRUD / キャッシュ参照
│   └── YouTubeClient.php       # YouTube API 呼び出し
├── assets/
│   ├── css/
│   │   └── style.css           # 画面スタイル
│   └── js/
│       └── app.js              # フロント制御・Chart.js 描画
├── sql/
│   └── schema.sql              # MySQL テーブル定義
└── tests/
    ├── bootstrap.php
    ├── GrokClientTest.php
    ├── SentimentAnalyzerTest.php
    └── YouTubeClientTest.php
```

### 主要ファイルの役割

| ファイル | 役割 |
|---|---|
| `index.php` | 入力フォーム・結果表示・フロント初期化 |
| `api/analyze.php` | リクエスト受信、バリデーション、分析実行、JSON 応答 |
| `src/YouTubeClient.php` | URL / 動画 ID 解釈、コメント取得 |
| `src/GrokClient.php` | コメント前処理後に Grok API へ送信 |
| `src/SentimentAnalyzer.php` | スコア集計と表示用データ生成 |
| `src/VideoRepository.php` | `videos` / `analysis_results` / `comments` の保存・検索 |
| `tests/` | PHPUnit テストコード |

## 4. デプロイ対象

ロリポップへ配置する対象は `php/` 配下である。  
詳細手順は [12_デプロイ手順書.md](12_デプロイ手順書.md) を参照すること。

### アップロード対象

```text
php/
├── index.php
├── api/
├── src/
├── assets/
├── sql/                # 参照用。標準は phpMyAdmin で schema.sql を実行
├── config.php          # 手動作成・手動アップロード
├── .htaccess
└── composer.json
```

### 本番不要

| パス | 理由 |
|---|---|
| `tests/` | テストコードのため |
| `phpunit.xml` | テスト設定のため |
| `config.php.example` | サンプルのみで実行不要 |
| `vendor/` | サーバー側で `composer install --no-dev` 可能な場合は不要 |

## 6. アクセス制御対象

Web から直接参照させないパスは以下の通り。

- `php/config.php`
- `php/src/`
- `php/sql/`

これらは `.htaccess` により遮断する。
