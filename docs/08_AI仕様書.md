# AI仕様書

YouTube動画コメント感情分析システム - Gemini API 感情分析エンジン

## 1. 概要

本システムは Google の **Gemini API** を使用して YouTube コメントの感情分析を実行する。
BERTのようなローカルモデルは使用せず、API経由でLLMに感情判定を委託する方式を採用している。

### 1.1 使用モデル

| 項目 | 値 |
|------|-----|
| APIプロバイダー | Google（Gemini） |
| APIエンドポイント | `https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent` |
| デフォルトモデル | `gemini-2.5-flash` |
| 設定変更 | `config.php` の `GEMINI_MODEL` で切り替え可能 |
| 認証方式 | `x-goog-api-key` ヘッダー |

### 1.2 利用可能モデル

| モデルID | 特徴 |
|---------|------|
| `gemini-2.5-flash` | 軽量・高速・無料枠向け（推奨） |
| `gemini-2.5-pro` | 高精度・上位モデル |

### 1.3 分類タスク

- **最終出力**: 3クラスの確率スコア（合計 1.0）
  - `positive`（0.0〜1.0）: ポジティブな感情
  - `negative`（0.0〜1.0）: ネガティブな感情
  - `neutral`（0.0〜1.0）: 中立または判定困難
- **感情ラベル**: スコアが最大のクラスを採用（`pos` / `neg` / `neutral`）

## 2. 処理フロー

```
コメント配列（最大100件）
  ↓
GeminiClient::analyzeSentimentBatch()
  ↓
20件ずつバッチ分割（BATCH_SIZE = 20）
  ↓
各バッチ: GeminiClient::analyzeBatch()
  ├── プロンプト生成（番号付きコメント一覧）
  ├── API呼び出し（POST :generateContent）
  └── JSONレスポンス解析・正規化
  ↓
SentimentAnalyzer::analyzeComments()
  ├── 各コメントにスコアを付与
  └── 感情ラベル（pos/neg/neutral）を決定
  ↓
SentimentAnalyzer::summarize()
  └── 集計（件数・比率・平均スコア）
```

## 3. APIリクエスト仕様

### 3.1 エンドポイント

```
POST https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent
```

`{model}` は `config.php` の `GEMINI_MODEL` で設定した値（デフォルト: `gemini-2.5-flash`）。

### 3.2 リクエストヘッダー

```
Content-Type: application/json
x-goog-api-key: {GEMINI_API_KEY}
```

### 3.3 リクエストボディ

```json
{
  "contents": [
    {
      "role": "user",
      "parts": [
        {
          "text": "Analyze the sentiment of each YouTube comment below.\n\nComments:\n1. コメントテキスト1\n2. コメントテキスト2\n...\n\nReturn only a JSON array. Do not include Markdown fences or explanations.\nEach item must have positive, negative, and neutral scores, and the three scores must sum to 1.0.\nExample:\n[{\"positive\":0.0,\"negative\":0.0,\"neutral\":0.0}]"
        }
      ]
    }
  ],
  "generationConfig": {
    "temperature": 0
  }
}
```

| パラメータ | 値 | 理由 |
|-----------|-----|------|
| `temperature` | 0 | 毎回同じ結果を得るため（決定論的） |
| `model` | config.phpで設定 | 環境により変更可能 |

### 3.4 レスポンス

```json
{
  "candidates": [
    {
      "content": {
        "parts": [
          {
            "text": "[{\"positive\":0.85,\"negative\":0.10,\"neutral\":0.05},{\"positive\":0.05,\"negative\":0.90,\"neutral\":0.05}]"
          }
        ]
      }
    }
  ]
}
```

## 4. バッチ処理設計

### 4.1 バッチサイズ

| 項目 | 値 |
|------|-----|
| バッチサイズ | 20件/リクエスト（`GeminiClient::BATCH_SIZE`） |
| タイムアウト | 60秒/リクエスト |
| 最大コメント前処理長 | 300文字（HTMLタグ除去後） |

### 4.2 バッチ分割例

コメント50件の場合:
- バッチ1: コメント1〜20（1回のAPIコール）
- バッチ2: コメント21〜40（1回のAPIコール）
- バッチ3: コメント41〜50（1回のAPIコール）

**APIコール数 = ceil(コメント数 / 20)**

## 5. プロンプト設計

```
Analyze the sentiment of each YouTube comment below.

Comments:
1. {comment_1}
2. {comment_2}
...
N. {comment_N}

Return only a JSON array. Do not include Markdown fences or explanations.
Each item must have positive, negative, and neutral scores, and the three scores must sum to 1.0.
Example:
[{"positive":0.0,"negative":0.0,"neutral":0.0}]
```

**設計ポイント**:
- 英語でプロンプトを記述（Gemini は多言語対応のため英語指示でも日本語コメントを正確に分析する）
- `temperature: 0` で一貫した判定
- JSON配列のみを要求し、余分な説明文を排除
- コードブロック（` ``` `）も不要と明示

## 6. レスポンス解析

### 6.1 JSON抽出

モデルが余分なテキストを出力した場合に備え、正規表現でJSON配列部分を抽出する:

```php
preg_match('/\[[\s\S]*\]/u', $content, $matches);
```

### 6.2 スコア正規化

各スコアの値を 0.0〜1.0 に制限し、合計が 1.0 になるよう正規化する:

```php
$sum = $pos + $neg + $neu;
// 合計が0の場合は均等配分（0.33/0.33/0.34）
// 合計が0以上の場合は正規化
$result = [
    'positive' => round($pos / $sum, 4),
    'negative' => round($neg / $sum, 4),
    'neutral'  => round($neu / $sum, 4),
];
```

### 6.3 デフォルト値（異常時）

| 状況 | positive | negative | neutral |
|------|---------|---------|---------|
| JSON解析失敗 | 0.33 | 0.33 | 0.34 |
| スコア合計が0 | 0.33 | 0.33 | 0.34 |
| 件数不足（補完） | 0.33 | 0.33 | 0.34 |

---

## 7. 感情ラベル判定

`SentimentAnalyzer::determineSentiment()` でスコアが最大のクラスを採用する:

| 条件 | ラベル |
|------|--------|
| positive ≥ negative かつ positive ≥ neutral | `pos` |
| negative > positive かつ negative ≥ neutral | `neg` |
| それ以外 | `neutral` |

---

## 8. エラーハンドリング

| エラー | 対応 |
|--------|------|
| HTTP 429（レート制限） | `GeminiApiException` を送出。メッセージ「Gemini API rate limit reached. Please retry later.」 |
| HTTP 200以外 | `GeminiApiException` を送出（ステータスコード含む） |
| curl接続失敗 | `GeminiApiException` を送出（curlエラーメッセージ含む） |
| JSON解析失敗 | デフォルトスコア（0.33/0.33/0.34）で継続 |
| 件数不足 | デフォルトスコアで不足分を補完 |

---

## 9. パフォーマンス

| 条件 | 目安 |
|------|------|
| 20件分析（バッチ1回） | 約2〜5秒 |
| 60件分析（バッチ3回） | 約6〜15秒 |
| 100件分析（バッチ5回） | 約10〜25秒 |

レート制限に達した場合は処理が中断される（リトライは実装していない）。
