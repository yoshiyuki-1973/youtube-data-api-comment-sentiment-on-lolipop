# デプロイ手順書

本書は、ロリポップ共有サーバーへの **初回デプロイ** と **コード更新時の反映** を扱う。
運用・障害対応は [13_運用手順書.md](13_運用手順書.md) を参照すること。

---

## 1. 前提

- 配置対象は `php/` ディレクトリ配下である
- `config.php` は `config.php.example` を元にローカルで作成する
- ローカルで `composer install` を実行する場合、PHP 8.3 以上と Composer 2 が必要である
- 標準の DB 適用手順は phpMyAdmin を使う
- SSH / CLI が利用できる場合は代替手順として利用可能

---

## 2. 初回デプロイ

### 2.1 DB スキーマ適用

標準手順:

1. ロリポップ管理画面を開く
2. 「マイSQL」から phpMyAdmin を開く
3. 対象 DB を選択する
4. `php/sql/schema.sql` を実行する
5. `videos` / `analysis_results` / `comments` が作成されたことを確認する

CLI 代替手順:

```bash
mysql -h <DB_HOST> -u <DB_USER> -p <DB_NAME> < php/sql/schema.sql
```

### 2.2 設定ファイル作成

ローカルで `php/config.php` を作成する。

```php
return [
    'YOUTUBE_API_KEY' => 'YOUR_YOUTUBE_API_KEY',
    'GROK_API_KEY'    => 'YOUR_GROK_API_KEY',
    'GROK_MODEL'      => 'grok-3-mini',
    'DB_HOST'         => 'mysql***.lolipop.jp',
    'DB_NAME'         => 'your_db_name',
    'DB_USER'         => 'your_db_user',
    'DB_PASS'         => 'your_db_password',
];
```

> `config.php` は `.gitignore` 対象のため、Git ではなく手動で配置する。

### 2.3 ローカルで Composer 実行

アップロード前に `php/` ディレクトリで Composer を実行し、依存関係とオートローダーをローカルで確認する。
これは **事前確認手順** であり、本番反映時の標準手順は後述の **サーバー側で `composer install --no-dev` を実行する方法** とする。

前提条件:

- PHP 8.3 以上
- Composer 2

```bash
cd php
composer install
```

ローカルで `composer install` を実行しても、`vendor/` はそのまま本番へアップロードしない前提とする。
PHP バージョン不足または Composer 1 系では依存解決に失敗するため、その場合はローカルでの動的テストは未実施として扱う。

### 2.4 FTP アップロード

接続例:

| 項目 | 内容 |
|---|---|
| ホスト | `ftp.lolipop.jp` |
| ユーザー | ロリポップ FTP アカウント |
| パスワード | ロリポップ FTP パスワード |
| ポート | `21` |

アップロード先は `public_html/` または公開用サブディレクトリとする。

### 2.5 アップロード対象

```text
php/
├── index.php
├── api/
├── src/
├── assets/
├── sql/
├── config.php
├── .htaccess
└── composer.json
```

### 2.6 本番へ不要なもの

- `tests/`
- `phpunit.xml`
- `config.php.example`
- `vendor/` 。
  標準手順ではサーバー側で `composer install --no-dev` を実行するため、ローカル生成物はアップロードしない。

### 2.7 サーバー側で Composer 実行する場合

標準手順:

本番反映時は、アップロード後にサーバー側で Composer 2 を用いて `composer install --no-dev` を実行して依存関係を解決する。

```bash
ssh <user>@ssh.lolipop.jp
cd ~/public_html/<your-directory>/
composer install --no-dev
```

`composer.json` または `composer.lock` に変更がない更新では、必要に応じて `composer dump-autoload` のみでもよい。

## 3. コード更新時のデプロイ

### 3.1 通常の更新

1. ローカルで修正・テストする
2. 必要に応じて `php/` で `composer install` または `composer dump-autoload` を実行してローカル確認する
3. 変更ファイルを FTP でアップロードする
4. `composer.json` / `composer.lock` に変更がある場合は、サーバー側で `composer install --no-dev` を実行する
5. ブラウザで画面確認する

> ローカル確認で `composer install` を使う場合も、PHP 8.3 以上と Composer 2 を前提とする。

### 3.2 設定変更を含む更新

- `config.php` を変更した場合は、同ファイルも手動で再アップロードする
- API キーや DB 接続先変更後は画面から疎通確認する

### 3.3 スキーマ変更を含む更新

- `schema.sql` を直接上書きするだけでは DB は更新されない
- 変更内容に応じて phpMyAdmin または CLI で個別 SQL を適用する
- 破壊的変更前は必ずバックアップを取得する

---

## 4. デプロイ後確認

- 画面が表示される
- `api/analyze.php` が 500 にならない
- `config.php` に直接アクセスできない
- `src/` / `sql/` が公開されていない
- 実際に 1 件以上の分析が成功する
