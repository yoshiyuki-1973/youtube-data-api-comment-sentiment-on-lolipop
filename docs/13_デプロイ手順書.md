# デプロイ手順書

本書は、ロリポップ共有サーバーへ **SSHログインせずに** 初回デプロイと更新反映を行う手順をまとめる。

運用・障害対応は [14_運用手順書.md](14_運用手順書.md) を参照すること。

---

## 1. 前提

- 配置対象は `php/` ディレクトリ配下である
- ロリポップ管理画面、phpMyAdmin、FTP またはロリポップFTPを使用する
- サーバー上で `ssh`、`composer install`、`mysql` コマンドは実行しない
- `config.php` はローカルで作成し、Git ではなく手動でアップロードする
- DB スキーマ適用はロリポップ管理画面から phpMyAdmin で行う

---

## 2. 初回デプロイ

### 2.1 DB スキーマ適用

1. ロリポップ管理画面を開く
2. 「データベース」または「ユーザー専用ページ」から phpMyAdmin を開く
3. 対象 DB を選択する
4. 「SQL」タブを開く
5. `php/sql/schema.sql` の内容を貼り付けて実行する
6. `videos` / `analysis_results` / `comments` が作成されたことを確認する

### 2.2 設定ファイル作成

ローカルで `php/config.php.example` をコピーして `php/config.php` を作成する。

```bash
cp php/config.php.example php/config.php
```

`php/config.php` に以下を設定する。

```php
return [
    'YOUTUBE_API_KEY' => 'YOUR_YOUTUBE_API_KEY',
    'GEMINI_API_KEY'  => 'YOUR_GEMINI_API_KEY',
    'GEMINI_MODEL'    => 'gemini-2.5-flash',
    'DB_HOST'         => 'mysql***.lolipop.jp',
    'DB_NAME'         => 'your_db_name',
    'DB_USER'         => 'your_db_user',
    'DB_PASS'         => 'your_db_password',
    'DB_CHARSET'      => 'utf8mb4',
];
```

`config.php` は API キーと DB パスワードを含むため、公開リポジトリには含めない。

### 2.3 ローカル確認

ローカルでテストを実行する場合のみ、`php/` ディレクトリで Composer を実行する。

```bash
cd php
composer install
vendor/bin/phpunit --testdox
```

本番サーバーでは Composer を実行しない。現在のアプリ本体は Composer の実行時依存パッケージを使っていないため、`vendor/` は本番アップロード不要である。

将来、本番実行に必要な Composer パッケージを追加した場合は、ローカルで `composer install --no-dev` を実行して作成した `vendor/` を FTP でアップロードする。

### 2.4 FTP アップロード

FileZilla などの FTP クライアント、またはロリポップFTPを使用する。

| 項目 | 内容 |
|---|---|
| ホスト | `ftp.lolipop.jp` |
| ユーザー | ロリポップ FTP アカウント |
| パスワード | ロリポップ FTP パスワード |
| ポート | `21` |

アップロード先は `public_html/` または公開用サブディレクトリとする。

### 2.5 アップロード対象

`php/` の中身を公開ディレクトリへアップロードする。

```text
public_html/
├── index.php
├── api/
├── src/
├── assets/
├── sql/
├── config.php
└── composer.json
```

### 2.6 パーミッション設定

アップロード後、ロリポップFTPまたは FTP クライアントで以下のパーミッションを確認する。

| 対象 | パーミッション | 補足 |
|---|---:|---|
| `public_html/` / `api/` / `src/` / `assets/` などのディレクトリ | `705` | ロリポップ推奨値 |
| `index.php` / `api/analyze.php` / `src/*.php` | `604` | PHP ファイルは所有者のみ書き込み可 |
| `assets/css/*.css` / `assets/js/*.js` | `604` | 静的ファイルも所有者のみ書き込み可 |
| `.htaccess` を設置する場合 | `604` | 改ざん防止のためロリポップ推奨値に合わせる |
| `config.php` | `600` | API キーと DB パスワードを含むため、可能な限り所有者のみ読み書き可にする |
| `sql/*.sql` をアップロードする場合 | `600` | 公開不要な設計・初期化用ファイルとして扱う |

`config.php` を `600` にして画面が動作しない場合のみ、`604` に変更して再確認する。`777` や `666` は第三者が書き込めるため使用しない。

### 2.7 アップロード不要なもの

- `tests/`
- `phpunit.xml`
- `config.php.example`
- `composer.lock`
- `vendor/`

`sql/` は公開ディレクトリに置かなくても動作する。設計資料としてアップロードする場合は、ブラウザから直接見えない配置にするか、アップロード後に削除してもよい。

## 3. コード更新時のデプロイ

### 3.1 通常の更新

1. ローカルで修正と確認を行う
2. 変更したファイルだけを FTP で上書きアップロードする
3. ブラウザで画面表示と `api/analyze.php` の動作を確認する

### 3.2 設定変更を含む更新

- `config.php` を変更した場合は、同ファイルも手動で再アップロードする
- API キー、DB 接続先、モデル名を変更した後は、実際に 1 件以上の分析を実行して確認する

### 3.3 DB 変更を含む更新

- `schema.sql` を上書きアップロードするだけでは DB は更新されない
- 変更内容に応じて phpMyAdmin の「SQL」タブから必要な SQL を実行する
- 破壊的変更の前には、phpMyAdmin から必ずバックアップを取得する

---

## 4. デプロイ後確認

- トップページが表示される
- `config.php` に直接アクセスできない
- `src/` や `sql/` がブラウザから一覧表示されない
- ディレクトリが `705`、公開ファイルが `604`、`config.php` が `600` または動作確認済みの `604` になっている
- YouTube URL を入力して分析が成功する
- キャッシュ済み動画では 24 時間以内に再分析結果が返る
