# テスト仕様書

YouTube コメント感情分析システムの PHPUnit テスト方針とテストケースをまとめる。

## 1. テスト方針

- PHPUnit 10 を使用する
- 外部 I/O を伴う YouTube API / Grok API / MySQL はモックまたは事前条件付きで扱う
- ローカル開発環境で PHP 8.3 と Composer 2 を用いて `composer install` を実行し、PHPUnit 実行環境を準備する
- ビジネスロジック、スコア集計、URL 解釈を中心にテストする

---

## 2. テスト実行

### 2.1 セットアップ

```bash
cd php
php --version
composer --version
composer install
```

`vendor/bin/phpunit` は `composer install` 実行後に生成される。
このファイルが存在しない場合、または `composer install` が PHP / Composer の要件不足で失敗する場合、動的テストは未実施となり、レビュー結果は静的確認中心として扱う。

### 2.2 実行コマンド

```bash
# 全テスト実行
vendor/bin/phpunit --testdox

# 特定ファイルのみ実行
vendor/bin/phpunit tests/YouTubeClientTest.php --testdox

# 詳細出力
vendor/bin/phpunit -v
```

> 補足: `php/vendor/bin/phpunit` が存在しない環境では、回帰テスト結果までは確認できない。

## 5. モック設計

### 5.1 GrokClient のモック

```php
$mockGrokClient = $this->createMock(GrokClient::class);
$mockGrokClient->method('analyzeSentimentBatch')
    ->willReturn([
        ['positive' => 0.85, 'negative' => 0.10, 'neutral' => 0.05],
        ['positive' => 0.05, 'negative' => 0.90, 'neutral' => 0.05],
    ]);

$analyzer = new SentimentAnalyzer($mockGrokClient);
```

### 5.2 テストデータサンプル

$comments = [

## 6. テスト対象外

以下は外部依存が強いため、ユニットテストからは除外する。

| 対象 | 理由 |
|------|------|
| `YouTubeClient::fetchVideo()` | YouTube API との実通信が必要 |
| `YouTubeClient::fetchComments()` | YouTube API との実通信が必要 |
| `GrokClient::analyzeSentimentBatch()` | Grok API との実通信が必要 |
| `VideoRepository` | MySQL 接続が必要 |
| `Database` | MySQL 接続が必要 |

これらは実際の動作確認をブラウザ / phpMyAdmin で行う。
