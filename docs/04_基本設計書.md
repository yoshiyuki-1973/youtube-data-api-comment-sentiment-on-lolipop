# 基本設計書

YouTube動画コメント感情分析システム（PHP / ロリポップ版）

## 1. システム全体構成

```
ユーザー（ブラウザ）
  │
  ├── GET  index.php         → HTML + CSS + JavaScript を返す
  │
  └── POST api/analyze.php   → JSON レスポンスを返す
        │
        ├── VideoRepository::findCachedResult()
        │     └── MySQL（キャッシュあり → 即返却）
        │
        ├── キャッシュなし時:
        │     ├── YouTubeClient::fetchVideo()
        │     │     └── YouTube Data API v3 (videos.list)
        │     ├── YouTubeClient::fetchComments()
        │     │     └── YouTube Data API v3 (commentThreads.list)
        │     ├── GrokClient::analyzeSentimentBatch()
        │     │     └── Grok API (api.x.ai/v1/chat/completions)
        │     │           10件ずつバッチ処理
        │     ├── SentimentAnalyzer::analyzeComments()
        │     └── VideoRepository::saveResult()
        │           └── MySQL（結果を保存）
        │
        └── JSON レスポンス返却
              └── JavaScript（Chart.js ドーナツグラフ + コメント一覧）
```

## 2. ホスティング構成

### 2.1 サーバー環境

| 項目 | 内容 |
|------|------|
| サービス | ロリポップ 共有サーバー（ハイスピードプラン） |

### 2.2 アクセス保護

| パス | 保護方法 |
|------|---------|
| `src/` | `.htaccess` で直接HTTPアクセス禁止 |
| `sql/` | `.htaccess` で直接HTTPアクセス禁止 |
| `config.php` | `.htaccess` `<Files>` ディレクティブで禁止 |

## 3. 処理フロー

### 3.1 初回分析フロー（キャッシュなし）

```
1. ユーザーが動画URL/IDとコメント件数を入力
2. JavaScript が POST /api/analyze.php にリクエスト送信
3. analyze.php が YouTubeClient::extractVideoId() で動画IDを抽出
4. VideoRepository::findCachedResult() でキャッシュを検索
   → キャッシュなし（24時間以内のレコードが存在しない）
5. YouTubeClient::fetchVideo() で動画メタデータ取得（1ユニット）
6. YouTubeClient::fetchComments() でコメント取得（1ユニット/100件）
7. GrokClient::analyzeSentimentBatch() で感情分析
   → 10件ずつバッチ分割 → Grok API 呼び出し
8. SentimentAnalyzer::analyzeComments() でスコアと感情ラベルを付与
9. SentimentAnalyzer::summarize() で集計
10. VideoRepository::saveResult() で MySQL に保存
11. JSON レスポンスを返却
12. JavaScript がグラフとコメント一覧を描画
```

### 3.2 キャッシュヒットフロー

```
1. ユーザーが動画URL/IDとコメント件数を入力
2. JavaScript が POST /api/analyze.php にリクエスト送信
3. analyze.php が動画IDを抽出
4. VideoRepository::findCachedResult() でキャッシュを検索
   → キャッシュあり（24時間以内のレコードが存在する）
5. MySQL から結果を取得
6. JSON レスポンスを返却（YouTube API / Grok API の呼び出しなし）
7. JavaScript がグラフとコメント一覧を描画
```

## 5. エラーハンドリング設計

### 5.1 カスタム例外階層

```
\Exception
  ├── YouTubeApiException（基底）
  │     ├── QuotaExceededException
  │     ├── AuthenticationException
  │     ├── VideoNotFoundException
  │     └── CommentsDisabledException

```

### 5.2 HTTP ステータスコードマッピング

| 例外クラス | HTTPステータス | レスポンス |
|-----------|--------------|------------|
| `\InvalidArgumentException` | 400 | `error` に例外メッセージを返す |
| `QuotaExceededException` | 429 | `error` に例外メッセージを返す |
| `AuthenticationException` | 403 | `error` に例外メッセージを返す |
| `VideoNotFoundException` | 404 | `error` に例外メッセージを返す |
| `CommentsDisabledException` | 422 | `error` に例外メッセージを返す |
| `GrokApiException` | 502 | 固定メッセージを返す |
| その他の例外 | 500 | 汎用メッセージを返す |

## 6. フロントエンド設計

### 6.1 画面構成

| セクション | 内容 |
|-----------|------|
| 入力フォーム | 動画URL/ID入力、コメント件数セレクト（10/30/50/100）、分析ボタン |
| ローディング | スピナー表示（分析中） |
| 動画情報 | タイトル、チャンネル名、再生回数、いいね数、コメント数 |
| 感情グラフ | Chart.js 4.4.0 ドーナツグラフ（ポジ/ネガ/ニュートラル比率） |
| 感情サマリー | 件数・比率・平均スコアの数値表示 |
| コメント一覧 | フィルタタブ（すべて/ポジ/ネガ/ニュートラル）+ コメントカード |

> UI では操作を単純化するため、コメント件数は `10 / 30 / 50 / 100` の固定選択肢を提供する。
> 一方でサーバー側 API はバリデーション上 `1〜100` を受け付け、`max(1, min(100, ...))` で丸める。

### 6.2 AJAX 通信仕様

**リクエスト**

```javascript
POST /api/analyze.php
Content-Type: application/json

{
  "url": "https://www.youtube.com/watch?v=VIDEO_ID",
  "limit": 50
}
```

> API の正式なフィールド名は `url` と `limit`。`input` / `comment_limit` は旧表記。

**レスポンス（成功）**

```json
{
  "success": true,
  "cached": false,
  "video": { ... },
  "summary": { ... },
  "comments": [ ... ]
}
```

**レスポンス（エラー）**

```json
{
  "success": false,
  "error": "エラーメッセージ"
}
```

## 7. MySQL キャッシュ設計

### 7.1 キャッシュ判定ロジック

```sql
SELECT ar.*, v.*
FROM analysis_results ar
JOIN videos v ON v.video_id = ar.video_id
WHERE ar.video_id = :video_id
  AND ar.comment_limit = :comment_limit
  AND ar.analyzed_at > DATE_SUB(NOW(), INTERVAL 24 HOUR)
ORDER BY ar.analyzed_at DESC
LIMIT 1
```

### 7.2 テーブル構成

| テーブル | 役割 |
|---------|------|
| `videos` | 動画メタデータ（1動画1レコード） |
| `analysis_results` | 分析サマリー（動画ID×件数をキャッシュ検索キーとして保持） |
| `comments` | 個別コメントのスコア（analysis_id に紐づく） |

> `analysis_results` の `video_id × comment_limit` は **一意制約ではない**。
> 同じ組み合わせの履歴が複数行存在しうる前提で、キャッシュ判定時は `analyzed_at` が新しいレコードを優先して参照する。

詳細は [07_テーブルレイアウト.md](07_テーブルレイアウト.md) を参照。
