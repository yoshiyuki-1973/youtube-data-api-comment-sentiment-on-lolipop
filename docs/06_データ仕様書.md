# データ仕様書

YouTube動画コメント感情分析システム（PHP / ロリポップ版）

## 1. データ方針

本システムでは、分析結果を MySQL（ロリポップ マイSQL）に **24時間キャッシュ** する。
同じ動画ID × コメント件数の組み合わせで24時間以内に再アクセスした場合は、YouTube API / Gemini API を呼び出さずにキャッシュから返す。

---

## 2. データフロー

```
ブラウザ
  │
  │ POST /api/analyze.php
  │ { "url": "YouTube URL または動画ID", "limit": 50 }
  ▼
VideoRepository::findCachedResult()
  ├── キャッシュあり → MySQL から取得 → JSON レスポンス（cached: true）
  │
  └── キャッシュなし
        │
        ├── YouTubeClient → YouTube Data API
        │     ├── videos.list → 動画メタデータ
        │     └── commentThreads.list → コメント一覧
        │
        ├── GeminiClient → Gemini API（20件/バッチ）
        │     └── 感情スコア配列 [positive, negative, neutral]
        │
        ├── SentimentAnalyzer
        │     ├── analyzeComments() → 各コメントにスコア付与
        │     └── summarize() → 集計結果
        │
        ├── VideoRepository::saveResult() → MySQL に保存
        │
        └── JSON レスポンス（cached: false） → ブラウザ
```

---

## 3. データ構造

### 3.1 API リクエスト（ブラウザ → api/analyze.php）

```json
{
  "url": "https://www.youtube.com/watch?v=dQw4w9WgXcQ",
  "limit": 50
}
```

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| `url` | string | 必須 | YouTube URL または 11文字の動画ID |
| `limit` | int | 任意 | 取得件数（1〜100、省略時は10） |

> フィールド名は **`url`** と **`limit`**。`input` / `comment_limit` は誤り。
> UI では `10 / 30 / 50 / 100` の固定選択肢を表示するが、API は内部的に `1〜100` を受け付ける。

---

### 3.2 動画情報（YouTubeClient::fetchVideo の戻り値）

```json
{
  "video_id":      "dQw4w9WgXcQ",
  "title":         "Never Gonna Give You Up",
  "channel_name":  "Rick Astley",
  "channel_id":    "UCuAXFkgsw1L7xaCfnd5JJOw",
  "published_at":  "2009-10-25T06:57:33+00:00",
  "view_count":    1500000000,
  "like_count":    17000000,
  "comment_count": 2000000
}
```

---

### 3.3 コメント情報（YouTubeClient::fetchComments の戻り値の各要素）

```json
{
  "text":         "最高の動画でした！",
  "author":       "ユーザー名",
  "like_count":   42,
  "published_at": "2025-01-01T00:00:00+00:00"
}
```

> フィールド名は **`author`**。`author_name` は誤り。

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `text` | string | コメント本文（YouTube API の `textDisplay`。HTMLタグを含みうる） |
| `author` | string | 投稿者表示名 |
| `like_count` | int | コメントのいいね数 |
| `published_at` | string | 投稿日時（ISO 8601） |

> 感情分析時は `text` に対して HTML タグ除去を行い、先頭300文字までをプロンプトに使用する。
> ただし API レスポンスおよび保存対象のコメント本文は `textDisplay` ベースの値を保持する。

---

### 3.4 感情スコア（GeminiClient の戻り値の各要素）

```json
{
  "positive": 0.85,
  "negative": 0.10,
  "neutral":  0.05
}
```

**制約**: `positive + negative + neutral = 1.0`（正規化済み）

---

### 3.5 感情分析済みコメント（SentimentAnalyzer::analyzeComments の戻り値の各要素）

```json
{
  "text":            "最高の動画でした！",
  "author":          "ユーザー名",
  "like_count":      42,
  "published_at":    "2025-01-01T00:00:00+00:00",
  "positive_score":  0.85,
  "negative_score":  0.10,
  "neutral_score":   0.05,
  "sentiment":       "pos"
}
```

---

### 3.6 集計結果（SentimentAnalyzer::summarize の戻り値）

```json
{
  "positive_count":     30,
  "negative_count":     12,
  "neutral_count":       8,
  "total_count":        50,
  "positive_ratio":      0.6,
  "negative_ratio":      0.24,
  "neutral_ratio":       0.16,
  "avg_positive_score":  0.6823,
  "avg_negative_score":  0.2104,
  "avg_neutral_score":   0.1073
}
```

> 集計キーは **`avg_positive_score`** 系。`avg_positive` は誤り。

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `positive_count` | int | `sentiment = 'pos'` のコメント件数 |
| `negative_count` | int | `sentiment = 'neg'` のコメント件数 |
| `neutral_count` | int | `sentiment = 'neutral'` のコメント件数 |
| `total_count` | int | 分析対象コメント総数 |
| `positive_ratio` | float | positive_count / total_count（4桁精度） |
| `negative_ratio` | float | negative_count / total_count（4桁精度） |
| `neutral_ratio` | float | neutral_count / total_count（4桁精度） |
| `avg_positive_score` | float | 全コメントの平均ポジティブスコア（4桁精度） |
| `avg_negative_score` | float | 全コメントの平均ネガティブスコア（4桁精度） |
| `avg_neutral_score` | float | 全コメントの平均ニュートラルスコア（4桁精度） |

---

### 3.7 API レスポンス（api/analyze.php → ブラウザ）

**成功時**:
```json
{
  "success": true,
  "cached": false,
  "video": {
    "video_id":      "string",
    "title":         "string",
    "channel_name":  "string",
    "view_count":    0,
    "like_count":    0,
    "comment_count": 0
  },
  "summary": {
    "positive_count":     0,
    "negative_count":     0,
    "neutral_count":      0,
    "total_count":        0,
    "positive_ratio":     0.0,
    "negative_ratio":     0.0,
    "neutral_ratio":      0.0,
    "avg_positive_score": 0.0,
    "avg_negative_score": 0.0,
    "avg_neutral_score":  0.0
  },
  "comments": [
    {
      "text":           "string",
      "author":         "string",
      "like_count":     0,
      "published_at":   "string",
      "sentiment":      "pos",
      "positive_score": 0.0,
      "negative_score": 0.0,
      "neutral_score":  0.0
    }
  ]
}
```

> `cached: true` の場合はキャッシュから返却（YouTube API / Gemini API 呼び出しなし）。

**エラー時**（`code` フィールドは存在しない）:
```json
{
  "success": false,
  "error": "エラーメッセージ（日本語）"
}
```

**HTTP ステータスコードマッピング**:

| 条件 | HTTP | エラーメッセージ |
|------|------|----------------|
| URL/ID 解析失敗 | 400 | 例外メッセージそのまま |
| 動画が見つからない | 404 | 例外メッセージそのまま |
| APIクォータ超過 | 429 | 例外メッセージそのまま |
| APIキー認証失敗 | 403 | 例外メッセージそのまま |
| コメント無効化 | 422 | 例外メッセージそのまま |
| Gemini API エラー | 502 | 「AI分析サービスへの接続に失敗しました。しばらく待ってから再試行してください。」 |
| その他の例外 | 500 | 「サーバーエラーが発生しました。」 |

## 4. MySQL データ構造

テーブル定義の詳細は [07_テーブルレイアウト.md](07_テーブルレイアウト.md) を参照。

### 4.1 テーブル概要

| テーブル | 主キー | 用途 |
|---------|--------|------|
| `videos` | `id` (AUTO_INCREMENT) | 動画メタデータ（`video_id` UNIQUE） |
| `analysis_results` | `id` (AUTO_INCREMENT) | 分析サマリー（`video_id × comment_limit` を検索キーとして利用） |
| `comments` | `id` (AUTO_INCREMENT) | 個別コメントスコア（`analysis_id` FK） |

### 4.2 キャッシュキー

| フィールド | 説明 |
|-----------|------|
| `video_id` | YouTube 動画ID |
| `comment_limit` | 取得コメント件数 |
| `analyzed_at` | 分析実行日時（24時間以内かで判定） |

> `video_id × comment_limit` は **UNIQUE 制約ではなくキャッシュ検索キー**。
> 同じ組み合わせの履歴が複数あってもよく、参照時は `analyzed_at` の新しいレコードを採用する。

## 5. YouTube Data API フィールドマッピング

### 5.1 videos.list

| PHPフィールド | YouTube API フィールド |
|-------------|----------------------|
| `video_id` | `id` |
| `title` | `snippet.title` |
| `channel_name` | `snippet.channelTitle` |
| `channel_id` | `snippet.channelId` |
| `published_at` | `snippet.publishedAt` |
| `view_count` | `statistics.viewCount` |
| `like_count` | `statistics.likeCount` |
| `comment_count` | `statistics.commentCount` |

### 5.2 commentThreads.list

| PHPフィールド | YouTube API フィールド |
|-------------|----------------------|
| `text` | `snippet.topLevelComment.snippet.textDisplay` |
| `author` | `snippet.topLevelComment.snippet.authorDisplayName` |
| `like_count` | `snippet.topLevelComment.snippet.likeCount` |
| `published_at` | `snippet.topLevelComment.snippet.publishedAt` |
